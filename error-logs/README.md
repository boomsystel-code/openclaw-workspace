# 错误日志索引

> 每次错误都是学习机会！所有错误都应该被记录、分析、避免重复。

## 文件结构

- `errors/YYYY-MM-DD/` - 每日错误记录
  - `summary.md` - 当日错误汇总
  - `error-XXX.md` - 具体错误详情

- `analysis/` - 错误分析报告
  - `weekly-summary.md` - 每周分析
  - `monthly-review.md` - 每月回顾
  - `patterns.md` - 错误模式识别

- `learnings/` - 经验教训库
  - `by-category/` - 按类别分类
  - `action-items.md` - 行动项

- `statistics/` - 统计数据
  - `metrics.json` - 量化指标
  - `trends.md` - 趋势分析

## 使用方法

### 1. 记录新错误
在memory/YYYY-MM-DD.md中添加错误记录，或在错误发生后立即创建新文件。

### 2. 分析错误
每周运行分析，更新patterns.md和learnings/。

### 3. 追踪改进
在action-items.md中记录改进措施，并在后续检查完成情况。

## 核心原则

1. **立即记录** - 错误发生后5分钟内记录
2. **完整上下文** - 错误信息 + 场景 + 解决方案
3. **提炼教训** - 从错误中提取可操作的建议
4. **避免重复** - 定期回顾，确保不犯同样错误
5. **持续改进** - 每错误都有改进机会

## 量化目标

- 错误重复率: < 5%
- 平均解决时间: < 30分钟
- 错误记录完整率: 100%
