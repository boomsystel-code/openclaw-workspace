## 🚀 OpenClaw 使用技巧 {#openclaw}

### 📚 5 个让 OpenClaw 能力提升 100 倍的实用技巧

来源：B站 AI技术博主（2026-02-07 学习）

#### 1️⃣ 开启记忆功能
- **配置**：
  - `compaction.memoryFlush.enabled = true`（记忆刷新）
  - `memorySearch.experimental.sessionMemory = true`（会话记忆搜索）
- **效果**：对话压缩前保留重点内容，可回溯整个聊天历史
- **适用场景**：长任务、复杂项目

#### 2️⃣ 任务配不同模型
- **原则**：不要所有事都用最强模型
- **分工**：
  - 编程 → 专用编码模型（如 Claude Code）
  - 信息检索 → 擅长检索的模型
  - 社交平台 → 懂社交语境的模型
- **效果**：结果更准确 + 成本更低

#### 3️⃣ 一次性告诉它关于你的信息
- **内容**：
  - 你的目标
  - 工作方式
  - 关注重点
  - 长期计划
  - 对它的期望（主动发现问题、不在线时继续推进）
- **效果**：它不再只是等指令，而是主动找事情做

#### 4️⃣ 反向提问
- **示例**：
  - "根据对我的了解，现在最值得做的事是什么？"
  - "我还能提供哪些信息能让你做得更好？"
- **效果**：把规划交给擅长规划的一方，人只确认方向

#### 5️⃣ 让 OpenClaw 自己动手做工具
- **示例**：
  - 任务看板
  - 记录系统
  - 对话回顾页面
- **问法**："我们还能做什么来提高效率？"
- **效果**：长期使用后拥有一套完全围绕自己工作方式搭建的系统

### 📚 学习资源

| 资源 | 链接 | 描述 |

| **Chrome Dev** | https://developer.chrome.com/docs | Chrome 开发者文档 |

### 🎯 核心技能领域

#### 1. 前端开发
- **HTML5**: 语义化标签、结构化内容
- **CSS3**: Flexbox/Grid 布局、动画、响应式设计
- **JavaScript(ES6+)**: 异步编程、DOM操作、模块化

#### 2. 性能优化
- **Core Web Vitals**: LCP, FID, CLS
- **代码分割**: 懒加载、Tree Shaking
- **缓存策略**: Service Worker, PWA
- **资源优化**: 图片压缩、字体优化

#### 3. 渐进式 Web 应用 (PWA)
- **离线访问**: Service Worker, Cache API
- **推送通知**: Web Push API
- **安装能力**: Web App Manifest
- **接近原生体验**: 后台同步、后台播放

#### 4. 现代框架
- **React/Vue/Angular**: 组件化开发
- **Next.js/Nuxt**: SSR/SSG, 文件路由
- **状态管理**: Redux, Jotai, Pinia
- **构建工具**: Vite, Turbopack

#### 5. 后端与 API
- **RESTful API**: 设计原则、版本控制
- **GraphQL**: 查询语言、Schema 设计
- **认证授权**: JWT, OAuth 2.0, Session
- **数据库**: ORM (Prisma, Drizzle), 索引优化

#### 6. 安全最佳实践
- **输入验证**: 防止 XSS, SQL 注入
- **HTTPS**: 证书管理, HSTS
- **CORS**: 跨域策略
- **Content Security Policy**: CSP 头配置

#### 7. 无障碍访问 (a11y)
- **WCAG 2.1**: 遵循 AA 级标准
- **语义化 HTML**: ARIA 标签
- **键盘导航**: Tab 顺序, 焦点管理
- **屏幕阅读器**: 替代文本, 语义结构
